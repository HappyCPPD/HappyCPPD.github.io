Lab Notes: Building an Asynchronous Chatroom with WebSockets

Python WebSockets Networking Asyncio

The Goal

As part of my journey into cybersecurity, I wanted to really understand how data moves back and forth in real-time. Before I can secure a network, I need to know how to build one. My goal for this project was to create a functioning client-server chat application from scratch using Python.

The Architecture and Logic

I decided to use WebSockets instead of standard TCP sockets because they are great for continuous, two-way communication. I mapped out the logic into two main pieces:

The Server (main.py): Acts as the central hub. It needs to keep a running list (a set()) of everyone who connects. When someone sends a message, the server catches it and loops through the set to broadcast that message to everyone else.

The Client (client.py): Connects to the server and handles the user interface. It needs to do two things at the exact same time: listen for new messages from the server, and wait for the user to type something on their keyboard.

The Roadblocks: Concurrency and Blocking

The biggest challenge I hit was getting the client to do two things at once.

In Python, the input() function is "blocking." That means if the program is waiting for me to type a message, it completely freezes. It won't print incoming messages from other people until I press enter.

I knew the logical solution was concurrency (running tasks simultaneously), but Python's asyncio library syntax can get complicated. I used AI-assisted coding to help me figure out the exact syntax needed. The solution was using asyncio.gather() to run my send and receive functions at the same time, and using loop.run_in_executor to push the blocking keyboard input into a background thread.

I acted as the architect, figuring out why it was breaking, and used AI to help me write the specific asynchronous boilerplate to fix it.

The Takeaway

Getting the chatroom working was a huge win. It proved I could design a stable client-server architecture and handle asynchronous data flows.

Right now, all the messages are sent in plain text. For a cybersecurity student, that's a big red flag. My next step for this project is to build an encryption layer on top of this foundation so the data is scrambled before it ever hits the WebSocket.
